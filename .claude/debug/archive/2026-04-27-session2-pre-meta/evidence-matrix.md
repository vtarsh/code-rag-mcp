- hit@5 = 35.79% / hit@10 = 41.63% / R@10 = 7.05% / nDCG@10 = 13.84%

- Avg `expected_paths` per query = 24.7, p50 = 13, p95 = 90, max = 471

## Cross-cutting empirical signals

1. **Wrong-repo failures dominate the floor**: 222/530 misses (42%) have NO GT repo in top-10 — pure repo-routing failure (H4, H10).
2. **Right-repo-wrong-file is the larger bucket**: 308/530 (58%) have GT repo in top-10. These need intra-repo file ranking improvement (H5/H8/H9).
3. **Index drift is the recall@10 ceiling, not the hit@10 floor**: ~42% of GT not indexed → R@10 capped at ~42% theoretical, hit@10 capped at 90.5%. Observed gap from 90.5% ceiling to 41.6% reality is the actual hit@10 deficit.
4. **Doc-tower routing dominates** (82.3%) but Hit@10 differential (3.7pp) is small. FTS5 leg compensates partially because chunks_fts is shared.
5. **Doc/markdown promotion bias is concrete**: docs file_type appears 29× more often at rank-1 in misses than in hits. Boost asymmetry (multiply pre-rerank vs subtract post-rerank) is a structural cause.

## Top-3 most diagnostic hypotheses (data-anchored)

- **H6** (index/eval drift) — highest impact on R@10 ceiling; 86 queries are unhittable. 100% empirical, no inference.
- **H3 + H9** (doc promotion via boosts compounded with skipped penalty) — explains 116/530 doc-top-1 misses and the 29× over-representation. Most likely intervention: disable boosts on jira A/B.
- **H4** (mis-routing) — 82.3% routed to docs tower; concrete classifier audit done.

## Hypotheses confidently DOWN-ranked

- **H2** (token count) — flat by token bucket; refuted as primary driver.
- **H7** (stratum gate) — fires on only 4.6% of queries; ≤1pp impact ceiling.
- **H8** (pool depth) — already swept; null per memory note + bench gap signature.
