## Detailed Inventory

### 1. Rules ‚Äî `.claude/rules/conventions.md`

- **Type:** Always-loaded generic rule set (14 lines).
- **Content:** Core project conventions:
  - Org-specific data lives in `profiles/{name}/`; zero hardcoded org names in `src/`.
  - Keep every search result ‚Äî deprioritize low-confidence but include them.
  - Improve base recall before tuning reranker; reranker is polish, not fix.
  - Ground truth = `repos_changed` from Jira; cross-validate with `files_changed` and `pr_urls`.
  - Recall over precision: false negatives are worse than false positives.
  - Benchmark baselines live in `profiles/{profile}/RECALL-TRACKER.md` ‚Äî never regress.
  - Parallel agents by default; serialize any agents that touch the same DB or files.
  - Priority: generic mechanisms (`src/`) > profile data > private profile repo.
  - Report what the tool found, not how full the context window is.
  - Crons either act on findings or exit silently ‚Äî no report-only jobs.
  - Build pipeline and data constraints: see [[.claude/docs/data-changes.md]].
  - Git push via `mcp__github__push_files` (owner: `vtarsh`, repo: `code-rag-mcp`).

### 2. Docs ‚Äî `.claude/docs/`

#### `.claude/docs/data-changes.md`
- **Type:** Build-pipeline reference.
- **Content:** Documents the full rebuild pipeline:
  ```
  extract_artifacts.py -> build_index.py -> build_graph.py -> build_vectors.py
  ```
  - Full rebuild ~2‚Äì4h, peaks ~20GB RAM.
  - `build_index.py` recreates repos/chunks tables ‚Äî backup `task_history` first.
  - Incremental: `build_index.py --incremental` re-indexes only changed repos (SHA comparison).
  - After adding tasks/references: `build_vectors.py --repos=task-slug,ref-slug`.
  - Full rebuild: `make build` or `ACTIVE_PROFILE=my-org ./scripts/full_update.sh --full`.
  - Constraints: FTS5 virtual tables cannot have columns added (use separate tables); restart daemon after editing glossaries; `analyze/` is a 13-module package ‚Äî add domains via `classifier.py` + new analyzer file.

#### `.claude/docs/workflow-cycles.md`
- **Type:** Workflow cycle reference.
- **Content:** Two cycles:
  1. **Task Validation Cycle** (recall/quality work):
     Collect ‚Üí Validate independently (no MCP RAG) ‚Üí Validate via MCP RAG ‚Üí Compare ‚Üí Improve ‚Üí Benchmark ‚Üí Repeat.
  2. **Continuous Improvement** (triggered after benchmark improvement, adding tasks, modifying cascade/classifier, every 10th deep analysis):
     Benchmark before/after ‚Üí Pattern mine (`/pattern-mine`) ‚Üí Implement if pattern found ‚Üí Benchmark again ‚Üí Update `RECALL-TRACKER.md` baselines.

#### `.claude/docs/development-pattern.md`
- **Type:** MCP development pattern guide.
- **Content:**
  - **Core rule:** Measure first, build second, validate third.
  - **Capability Ladder** (6 rungs, do not skip): Gotcha ‚Üí Process doc ‚Üí Convention ‚Üí Tool mode ‚Üí New tool ‚Üí Pipeline. Promotion requires 3+ sessions.
  - **Pre-Flight Checklist** (10 items) before any MCP improvement.
  - **What to Pre-compute vs Not:** Pre-compute topology, navigation pointers, co-change patterns, verified behavioral facts, process guides, recipes. Do **not** pre-compute field mappings, type definitions, validation schemas, status enums, gap conclusions.
  - **Nightly Refresh:** FTS5 index, dependency graph, vectors, benchmarks ‚Üí auto; gotchas/recipes/process docs ‚Üí manual; co-change rules ‚Üí weekly re-mine.
  - **Sync vs Async:** Always check if a change has an async path (webhook ‚Üí Temporal).
  - **Feedback Loop:** Session ‚Üí `tool_calls.jsonl` ‚Üí `analyze_session_quality.py` (weekly) ‚Üí identify struggling patterns ‚Üí write gotcha ‚Üí validate ‚Üí deploy.

### 3. Agents ‚Äî `.claude/agents/`

| File | Scope | Description |
|------|-------|-------------|
| `deep-analysis.md` | **Generic** | Deep Analysis Agent Instructions. Analyzes ONE Jira task independently (no MCP RAG) to find all repos that should be involved. Protocol: gather task context from `knowledge.db`, independent repo discovery (grep, graph edges, similar tasks), run `benchmark_recall.py`, compare & classify each expected repo with root-cause codes (phantom, no_keyword_signal, graph_gap, etc.), output structured JSON. |
| `pattern-miner.md` | **Generic** | Pattern Miner Agent Instructions. Analyzes `task_history` data to discover actionable patterns for improving recall/precision. Runs: co-change pairs (>80% conditional probability), never-alone repos, hub analysis (BFS fan-out), domain template validation (PI/CORE/BO/HS), missed repo analysis. Outputs structured JSON. |
| `pay-impact-audit.md` | **pay-com** | Step-Scoped Provider Audit Agent. Audits ONE step of a payment flow for a specific provider. Receives injected context (this step, previous step, next step, reference provider, task scope, reviewer constraints). Methodology: existence verification (grep before any HIGH+ finding), gather context, check provider docs, check integration checklist, check PR review patterns, trace cross-step impact, check platform generic handling, severity calibration (CRITICAL/HIGH/MEDIUM/LOW/INFORMATIONAL), anti-pattern checklist. Output structured JSON. |
| `pay-line-trace.md` | **pay-com** | Line Trace auditor (3 stages: `map-response`, `provider-request`, `webhook`). Line-by-line audit of a provider implementation at one stage. Reads `profiles/pay-com/docs/references/field-contracts.yaml`, `reference-snapshots.yaml`, `global-conventions.md`, reference implementations (volt / paysafe / trustly), and target provider files. Outputs one row per finding with FIELD, FILE, CHECK, EXPECTED, ACTUAL, REFERENCE, SEVERITY. |

### 4. Skills ‚Äî `.claude/skills/`

Each skill is a directory containing `SKILL.md`.

| Skill Directory | Description |
|-----------------|-------------|
| `collect-tasks/` | **Jira Task Collection.** Usage: `/collect-tasks [options]`. Collects tasks from Jira, enriches with GitHub PR data, verifies data shape. Primary tool: `profiles/pay-com/scripts/auto_collect.py`. Runs hourly via launchd. Post-collection verification SQL included. |
| `deep-analysis/` | **Deep Task Analysis.** Usage: `/deep-analysis <task-id>`. Classifies task tier (1‚Äì4), gathers context from `task_history`, runs independent discovery vs MCP RAG tool, compares & classifies, outputs structured JSON. Agent prompt: [[.claude/agents/deep-analysis.md]]. |
| `pattern-mine/` | **Pattern Mining from Task History.** Usage: `/pattern-mine [options]`. Launches pattern-miner agent ([[.claude/agents/pattern-miner.md]]) to run co-change pairs, never-alone repos, hub analysis, domain template validation, missed repo analysis. Triggered after benchmark improvements, new tasks, cascade changes, every 10th deep analysis. |
| `recall-test/` | **Recall Benchmark Suite.** Usage: `/recall-test [options]`. Runs four benchmarks in sequence: `benchmark_queries.py`, `benchmark_realworld.py`, `benchmark_flows.py`, `benchmark_recall.py`. Compares against baselines in `profiles/pay-com/RECALL-TRACKER.md`. Must run before and after any search/indexing changes. |
| `scrape-docs/` | **Provider Documentation Scraper.** Usage: `/scrape-docs <provider> [url]`. Fetches and indexes provider API docs. Open docs via Tavily (`profiles/pay-com/scripts/tavily-docs-crawler.py`) ‚Üí `.secrets/provider-docs/`, then copy to `profiles/pay-com/docs/providers/`. Closed docs (auth required) via browser extension + Playwright MCP. After fetching, index with `build_index.py` or `build_vectors.py`. |
