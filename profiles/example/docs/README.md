# Profile Documentation

Place organization-specific documentation here:

## flows/
YAML files describing request/data flows through your services.
These get indexed and searchable via the RAG system.

Example: `flows/auth-flow.yaml`

## gotchas/
Markdown files with known quirks, debugging tips, and institutional knowledge.
These get indexed as high-priority searchable content.

Example: `gotchas/database-migrations.md`
